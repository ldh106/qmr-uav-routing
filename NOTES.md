## 1️⃣ β(QL_B0p8) 실험 데이터 불일치 발견 및 수정

| | 내용 |
|---|---|
| 🔎 발견 | 원래 결과 파일에서 QL(베이스라인)과 QL_B0p8(변형)의 **"탐지된 이벤트 수"가 서로 달랐음** (베이스라인 50개 vs 변형 277개). 같은 환경에서 알고리즘만 다르게 실행했다면 이 값이 같아야 하는데, 다르다는 것은 두 결과가 애초에 다른 환경 설정에서 실행됐다는 뜻 |
| 🧩 원인 | 실험을 여러 시나리오(고기동/지연민감/혼잡)로 나눠 반복 실행하고 결과 파일명을 정리하는 과정에서, 혼잡 시나리오에서 실행한 QL_B0p8 결과 파일이 다른 시나리오 폴더에도 잘못 섞여 들어간 것으로 확인 |
| ✅ 수정 | "탐지된 이벤트 수가 두 그룹 간에 일치하는 결과만 비교 대상으로 채택한다"는 기준으로 원본 JSON 데이터 전체를 다시 필터링해 재계산 |

## 2️⃣ Two-Mode Policy에서 DM 페널티가 효과 없던 이유

URG(Two-Mode Policy)와 DM(deadline-miss penalty)을 같이 켰을 때 결과가 URG
단독과 완전히 동일하게 나오는 현상을 발견했습니다. `q_learning_routing.py`를
다시 살펴보니:

```python
max_ttl_steps = int(getattr(self.simulator, "event_duration", 2001))
```

긴급 모드 판단 기준(`ttl_frac`)이 패킷의 실제 전체 TTL(`packets_max_ttl`)이
아니라 이벤트 유효시간(`event_duration`, 훨씬 작은 값)을 기준으로 계산되고
있었습니다. 그 결과 대부분의 패킷이 이미 긴급 모드로 분류되어, DM 페널티 항이
긴급 모드 스코어(`urgency_gain` 항)에 묻혀 사실상 반영되지 않은 것으로
추정됩니다.

> ℹ️ 이 부분은 재구현하지 않고 원인 분석까지만 기록해 두었습니다.
